# Audit Field

```text
You are working in the Design System v2.0 2026 Figma file.

Goal:
Audit the new `field` component and confirm whether it is ready to support the code design-system pipeline.

Do not make changes in this pass. Only inspect and report.

Context:
The input primitive components now look structurally correct:
- text-input
- select-input
- number-input
- input-otp
- search-input

The remaining concern is the `field` component.

Please audit the `field` component for these points:

1. Variant API

Check whether `field` has a `required` variant.

If the component only has:
- `required=false`

then report that this is not useful as a variant property.

Expected result:
- Either add both `required=false` and `required=true` in the next fix pass
- Or remove `required` entirely if required labels are not part of this design system

2. Text token bindings

Inspect all visible text layers inside the `field` component, including:

- label
- description/help text
- error text
- disabled label text
- disabled description text

Check whether these are bound to semantic/component variables for:

- text color
- font family
- font size
- line height
- font weight

Expected direction:
- Field label should use the same typography and color variables as `.input/label`
- Field description/help text should use the same typography and color variables as `.input/description`
- Field error text should use destructive/error semantic variables
- Disabled field text should use disabled semantic foreground variables

3. Control slot

Inspect the dashed `control-slot`.

Confirm that it is only a placeholder for documentation/composition and not meant to become a real coded visual primitive.

Expected direction:
- The slot may exist in Figma as a placeholder
- In code, `ds-field` should contain real controls passed by consumers

4. Report format

Please report:

- What is already correct
- What is missing
- Which exact field layers need changes
- Whether each issue is a Figma variable binding issue, a variant API issue, or just documentation
```
